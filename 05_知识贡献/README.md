# 知识贡献

> 每天收工前，把今天学到的东西、写好的代码、整理好的资料，从项目文件夹复制一份到自己的目录里，然后 push 到 GitHub。
>
> 这就是"知识贡献"——**不只是代码，还有你为什么这么写、踩了什么坑、下一步怎么做。**

---

## 怎么用

### 每天早上
```bash
git pull    # 拉伍尚京昨晚整理的台账，看看其他人干了啥
```

### 每天晚上（收工前）
```bash
# 1. 把今天做的东西复制到自己的文件夹
cp -r ~/project/xxx   ./05_知识贡献/你的名字/

# 2. 写或更新 说明.md（今天干了啥、卡在哪、下一步）
#    —— 3-5 句话就行，不用写作文

# 3. 同步到 GitHub
git add .
git commit -m "知识贡献：你的名字 - 今天干了什么"
git push
```

### 说明.md 写什么（模板）

```markdown
# 我的交付说明

## 今天（2026-06-24）

**干了啥**：
- XXX 功能写完了
- XXX 问题排查了

**卡在哪**：
- XXX 报错还没搞定

**下一步**：
- 明天做 XXX

**文件清单**：
- `xxx.py` ← 检测脚本
- `xxx.md` ← 使用说明
```

写 3-5 句话就行，别搞成论文。

---

## ⚠️ 仓库纪律

**非常重要，每个人都要看清楚：**

1. **只能动自己的文件夹** — `05_知识贡献/你的名字/` 里面的东西随便改
2. **禁止改动仓库根目录和其他人的文件夹** — 你一 `git add .` 可能把整个项目文件都带上去，把别人的东西覆盖掉
3. **如果你不确定** — 只用 `git add 05_知识贡献/你的名字/` 这一条路径，不要用 `git add .`
4. **听话** — 真搞炸了仓库大家都没得用

```bash
# ✅ 正确做法
git add 05_知识贡献/你的名字/
git commit -m "知识贡献：xxx"
git push

# ❌ 不要这样
git add .
# 你不知道你加了什么进去
```

## 规则

1. **每天至少一条 commit** — 哪怕只写了三行笔记也算
2. **代码和文档都放** — .py 源文件 + .md 说明，缺一不可
3. **别人的文件夹可以看** — 但不许改，有问题群里说
4. **不要放编译产物** — .py 文件就够了，build 目录别 commit
5. **说明.md 用中文写** — 不用装逼写英文，自己人看得懂就行
6. **⚠️ 严禁 `git add .` 无脑提交** — 只 add 自己的文件夹路径
