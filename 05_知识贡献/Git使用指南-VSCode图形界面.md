# Git 使用指南（VS Code 图形界面版）

> **不用记命令，全用鼠标点。** 每天早晚各花 10 秒。

---

## 第一次：克隆仓库

只有第一次需要做，之后不用管了。

1. 打开 VS Code
2. 按 `Ctrl+Shift+P` → 输入 `git clone` 回车
3. 粘贴仓库地址 → 选择保存位置（建议放桌面或项目文件夹）
4. VS Code 右下角点"打开"

**以后每次打开 VS Code，直接点 文件 → 打开文件夹 → 选这个文件夹就行。**

---

## 每天早上：拉取最新

让 VS Code 读到你队友昨晚提交的东西：

1. 打开 VS Code 左侧的 **源代码管理** 图标（`Ctrl+Shift+G`）
2. 顶部的三个点 `...` → 点击
3. 选择 **拉取（Pull）**
4. 完成 ✅

> **更快的方式**：直接 `Ctrl+Shift+P` 搜 `git pull` 回车。

---

## 每天晚上：提交 + 推送

> **只提交 `05_知识贡献/你的名字/` 里面的东西，别碰别人的文件夹。**

### 第一步：把文件放进去

把你今天写的代码、笔记复制到 `05_知识贡献/你的名字/` 对应的位置。

### 第二步：暂存（Stage）

1. 打开源代码管理面板（`Ctrl+Shift+G`）
2. 你会看到"更改"列表——里面有你新加的文件
3. 鼠标移到你的文件上 → 点右侧的 **`+`** 号
   - **只加你自己文件夹里的文件**
   - 如果看到别人的文件，**不要点 +**
4. 加完后文件会移到"暂存的更改"下面

### 第三步：提交（Commit）

1. 在"消息"框里写一句话，比如：
   ```
   知识贡献：林悦超 - 今天装了 YOLO 环境
   ```
2. 点消息框上面的 **✓ 提交** 按钮
3. 会弹窗问你是否要暂存所有更改→ 选**否**，你已经手动加过了

### 第四步：推送（Push）

1. 点左下角的 `...` 或状态栏的同步图标
2. 选择 **推送（Push）**
3. 完成 ✅

## 日常工作流程总结

```
早上到实验室：
  Ctrl+Shift+P → git pull        ← 10秒

干了一天的活...

收工前：
  复制文件到 05_知识贡献/你的名字/
  Ctrl+Shift+G → 点 + 号 → 写消息 → ✓提交 → ...推送
                                    ← 30秒
```

**每天就这两步，没有别的了。**

## 常见问题

### Q：我点错了，加了别人的文件怎么办？
在"暂存的更改"里找到那个文件 → 鼠标移上去 → 点 `−` 号取消暂存。

### Q：提交完发现写错字了怎么办？
不影响，下次提交写对就行。

### Q：推送失败说"不是最新的"？
说明有人在你之前推送了。先 `pull` 一下，再重新提交+推送。

### Q：我不小心把整个仓库搞乱了怎么办？
**不要慌，不要乱点。** 在群里喊伍尚京。
