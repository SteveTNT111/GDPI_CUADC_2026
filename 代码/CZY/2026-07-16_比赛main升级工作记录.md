## 任务流程整理

比赛流程简化为：

1. 飞手确认后，飞机自动起飞。
2. 起飞后飞往投放区上空。
3. 到达投放区后进入搜索流程。
4. 投放区上方按 2 到 3 个扫描航点覆盖约 `8m x 5m` 投放区。
5. 搜索高度初值建议 `3.5m ~ 4.0m`。
6. 识别到桶后不立刻盲投，先建立目标候选列表。
7. 多帧稳定确认后选择最可信目标。
8. 瞄准时根据 A/B 抛投器做前后 5cm 补偿。
9. 对准稳定后执行抛投。
10. 投放完成后减少对应弹药，记录最近投放目标，避免重复锁定。
11. A/B 弹药耗尽后进入侦察流程。
12. 飞到侦察区，适当升高，短暂观察。
13. 发送 RTL，飞控执行返航/降落。

- NED / WGS84 坐标

- 稳定评分

本次实现的简化可跑策略：

- 订阅 `/vision/yolo/detections` 全量目标。
- 对目标做多帧近邻关联。
- 根据图像中心距离和 ENU 位置距离匹配。
- 连续出现达到 `target_min_hits` 后才认为有效。
- 短时间丢失不立刻删除。
- 优先选择位置有效、连续稳定、靠近中心、置信度较高的目标。
- 投放后将该目标 ENU 坐标加入黑名单，避免重复打同一个桶。

## 识别稳定和锁定策略整理

本次实现：

- 不直接用单帧 YOLO 框中心瞄准。
- 对目标 ENU 和机体系坐标维护最近若干帧样本。
- 使用中值稳定目标坐标。
- 如果目标跳变超过 `target_reject_jump_m`，拒绝该帧更新。

- 连续丢失超过 `aiming_lost_timeout_s` 后回到搜索。

## 飞控控制要求整理

本次实现遵循：

- 起飞、GUIDED、ARM、RTL、安全确认参考现有已验证代码。
- WGS84 全局航点使用 `/mavros/setpoint_raw/global`。
- 局部 setpoint 使用 `/mavros/setpoint_position/local`，遵守 MAVROS ENU 话题约定。
- `/mavros/local_position/pose` 作为 ROS ENU。
- 不混用 NED 和 ENU 轴。
- 模式切换后回读 `/mavros/state` 确认。
- 异常时空中优先 RTL，RTL 失败再 LAND。
- 地面测试和默认配置不会误解锁、误起飞、误开舵机。

## 抛投逻辑整理

本次实现：

- 初始弹药由 `ammo_a` / `ammo_b` 参数控制。
- 优先使用 A，A 用完后使用 B。
- A 抛投器前向偏移默认 `+0.05m`。
- B 抛投器后向偏移默认 `-0.05m`。
- 补偿方向按 detector_node 现有映射处理：机体 X 为前。
- 实投通过 `MAV_CMD_DO_SET_SERVO`。
- `enable_servo_drop=false` 时只 dry-run，不打开舵机。
- 投放后减少弹药，并通过 `/vision/mission_status` 显示 `A DROP!!!` 或 `B DROP!!!`。

投放条件：

- 飞控处于 `GUIDED`。
- main 状态是 `AIMING`。
- 目标位置有效。
- 水平误差小于 `aiming_threshold_m`。
- 稳定持续超过 `aiming_stable_time_s`。
- 距离上次投放超过 `drop_cooldown_s`。
- 当前抛投器还有弹药。

## 关键参数

飞行安全：

- `enable_takeoff`：是否允许自动 GUIDED/ARM/TAKEOFF，默认 `false`。
- `require_authorization`：是否要求飞手终端确认，默认 `true`。
- `authorization_phrase`：授权短语，默认 `YES`。
- `ground_test`：地面测试模式，不发送起飞、setpoint、实投。

抛投：

- `enable_servo_drop`：是否真的发送舵机命令，默认 `false`。
- `ammo_a` / `ammo_b`：A/B 初始弹药。
- `servo_a_channel` / `servo_b_channel`：A/B 舵机通道。

- `servo_hold_s`：舵机打开保持时间。

- `dropper_a_forward_offset_m`：A 抛投器相对相机前向偏移，默认 `+0.05`。

## 风险点和待实测项

- A/B 前后 5cm 补偿按 detector_node 现有映射处理：`X_body = -Y_cam`，机体 X 为前。仍需实测确认 D435i 实际安装方向与代码注释一致。
- 主状态机锁定目标 ENU 后不逐帧追检测框；若 GPS/local pose 漂移较大，锁定点会受影响。
- 3 个搜索航点覆盖 `8m x 5m` 只是初值，应根据实测 YOLO 高度漏检率调整。
- 搜索高度初值建议 `3.5m ~ 4.0m`，瞄准高度默认 `2.0m`，都需要实测修正。
- WGS84 区域入口只负责飞到入口点，搜索航线仍建议使用 local ENU 扫描点，避免多坐标系同时争夺控制权。
- `enable_servo_drop` 默认关闭，实投前必须单独验证舵机通道、PWM 开关方向和保持时间。
- 真机前必须确认 MAVROS 模式名、ArduPilot GUIDED 行为、RTL_ALT_FINAL 参数和 FAILSAFE 行为。
