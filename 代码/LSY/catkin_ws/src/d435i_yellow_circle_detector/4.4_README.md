# D435i 视觉检测 + 大地坐标变换 — src4.4 完整说明

> **版本：** src4.4(显示假的大地坐标) — 独立功能包，可单独拿到工作空间复现
> **ROS 包名：** `d435i_yellow_circle_detector`
> **系统：** Ubuntu 20.04 + ROS Noetic
> **相机：** Intel RealSense D435i，默认 30 FPS

旧版只能输出相机坐标系下的目标位置 (x, y, z)。本版新增 `target_geopose_node.py`，通过 tf2 + 四元数旋转 + geographiclib，将检测结果完整变换到 WGS84 大地坐标（经纬高），飞控可以直接用这个坐标导航。

> **版本来源：** 对应 `视觉组旧版本代码管理/src4.4(显示假的大地坐标)`。
> 本版只清理了一个未使用的 import（`tf2_geometry_msgs`），其余代码与 src4.4 完全一致。
> 迭代链路：4.0 → 4.1 → 4.2 → 4.3 → 4.3.1 → **4.4**

## 总体数据流

### YOLO 检测管线（比赛主用）

```
D435i → d435i_camera_node.py
  → /d435i/color/image_raw + /d435i/aligned_depth/image_raw + /d435i/color/camera_info
  → yolo_detector_node.py
  → /yolo/detection（最佳目标）+ /yolo/detections（全部目标）
```

### 大地坐标变换管线 ★

```
/yolo/detection                    → target_geopose_node.py
/mavros/global_position/global     → target_geopose_node.py  (无人机 GPS)
/mavros/local_position/pose        → target_geopose_node.py  (无人机 ENU 位姿)
                                      ↓
                               tf2: 相机系 → 机体坐标系
                               四元数旋转: 机体 → ENU(东北天)
                               geographiclib: ENU → WGS84 经纬高
                                      ↓
                              /competition/target_global (GeoTarget)
```

## 节点详解

### d435i_camera_node.py — 相机驱动

**功能：** 打开 D435i，发布彩色图 + 深度图 + 相机内参。默认 640×480 @ 30FPS。自动枚举并切换 RealSense profile 应对 USB 问题。

**发布话题：**

| 话题 | 类型 | 用途 |
|------|------|------|
| `/d435i/color/image_raw` | sensor_msgs/Image | 彩色图 |
| `/d435i/aligned_depth/image_raw` | sensor_msgs/Image | 对齐深度图 |
| `/d435i/color/camera_info` | sensor_msgs/CameraInfo | 相机内参 |

---

### yolo_detector_node.py ✏️ — YOLO 检测

**功能：** 加载 YOLOv8 模型逐帧推理，发布检测结果和标注图。本版新增了 geopose 数据订阅——在标注画面上叠加 GPS 经纬高和机体坐标。

**发布话题：**

| 话题 | 类型 | 说明 |
|------|------|------|
| `/yolo/detection` | YoloDetection | 最高置信度目标 |
| `/yolo/detections` | YoloDetections | 全部目标 |
| `/yolo/annotated_image` | sensor_msgs/Image | 标注画面 |

---

### target_geopose_node.py ★ — 坐标变换核心

**功能：** 将 YOLO 检测结果从相机系完整变换到 WGS84 大地坐标。

**坐标变换链：**
```
相机光学系 → tf2 → 机体坐标系 → 四元数旋转 → ENU(东北天) → geographiclib → 经纬高
```

**输入（需 MAVROS 运行）：**

| 话题 | 类型 | 说明 |
|------|------|------|
| `/yolo/detection` | YoloDetection | YOLO 最佳目标 |
| `/mavros/global_position/global` | NavSatFix | 无人机 GPS |
| `/mavros/local_position/pose` | PoseStamped | 无人机 ENU 位姿 |

**输出：** `/competition/target_global` (GeoTarget，含经纬高)

**状态码：**

| 状态 | 含义 |
|------|------|
| `ok` | 变换成功 |
| `no_detection` | 未检测到目标 |
| `low_confidence` | 置信度低于阈值 |
| `no_valid_global_position` | GPS 无固定解 |
| `tf_camera_to_body_failed` | tf2 变换失败 |

---

### competition_task_node.py — 比赛任务仲裁

**功能：** 根据 `mission_stage` 参数（drop / disaster）将 YOLO 结果翻译为比赛目标。

- `drop`：识别投放区白色圆筒，按 15/20/25cm 直径分类
- `disaster`：优先识别危险化学品标识（12cm），其次识别 20cm 圆筒


**输出：** `/competition/target`

---

## 消息定义

### GeoTarget.msg ★ 新增

```yaml
Header header | bool valid | string status
string source_topic / class_name
float64 confidence / center_x / center_y
float64 camera_x/y/z_m      # 相机系
float64 body_x/y/z_m        # 机体系
float64 enu_east/north/up_m # ENU 偏移
float64 latitude / longitude / altitude  # WGS84 大地坐标
```

## 常见问题

**D435i 不识别：** 插 USB 3.0 蓝色口，`realsense-viewer` 确认画面。日志中 `usb=2.1` 表示工作在 USB2 模式，需换 USB3 线或口。

**YOLO 报错：** `pip3 install ultralytics torch`

**TF 报错：** MAVROS 是否在运行？`rostopic list | grep mavros`

**geopose status 不是 ok：** 检查 MAVROS 是否提供 GPS（`/mavros/global_position/global`），GPS 是否已 fix。

**坐标不准：** 检查 `invert_camera_x` 参数，镜头朝下设 `true`。
