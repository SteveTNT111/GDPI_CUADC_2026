# 自主搜寻与双抛投任务——快速启动指南

这份文档用于让不了解代码的操作人员快速启动任务。

> 当前暂定：A 抛投器使用 CH5，B 抛投器使用 CH6。实际接线和 PWM 未确认前，禁止启用真实投放。

## 一、任务会做什么

```text
起飞到 2.5 m
→ 保持高度，以 0.20 m/s 向前搜索 3 m
→ 没找到目标：升到 3 m，自动返航降落
→ 找到目标：边下降到 1.2 m，边修正 X/Y 位置
→ X/Y 均在 ±0.20 m 内稳定 2 秒
→ 投放 A
→ 向前移动 0.08 m，稳定 2 秒
→ 投放 B
→ 升到 3 m，自动返航降落
```

坐标方向：

- `+X`：飞机向右
- `+Y`：飞机向前
- `+Z`：飞机升高

## 二、每次启动前检查

确认以下项目：

- 飞机位于空旷区域，前方至少有 5 m 安全距离。
- 遥控器已连接，操作人员能随时切换出 `GUIDED`。
- 桨叶、GPS、飞控、电池和 D435i 相机工作正常。
- 飞控串口是 `/dev/ttyACM0:921600`。
- 没有运行旧的 `main.py`、`competition_main.py` 或 `auto_drop_node.py`。
- 真实投放前，已经确认 A/B 舵机通道、PWM 和机械动作。

## 三、打开终端并加载环境

在 NUC 上打开一个终端，复制执行：

```bash
cd ~/catkin_ws
source /opt/ros/noetic/setup.bash
source devel/setup.bash
```

如果是第一次运行，或 ROS 找不到节点，可以先编译：

```bash
cd ~/catkin_ws
catkin_make --pkg cuadc_vision
source devel/setup.bash
```

## 四、先做安全预览

安全预览只检查配置，不连接飞机、不解锁、不起飞、不控制舵机：

```bash
roslaunch cuadc_vision autonomous_two_drop_mission.launch \
  start_mavros:=false \
  start_vision:=false
```

看到下面的提示表示预览正常：

```text
SAFE PREVIEW ONLY
```

## 五、正式启动整套任务

以下命令会启动 MAVROS、相机、YOLO 和自主任务，并可能真实解锁起飞：

```bash
roslaunch cuadc_vision autonomous_two_drop_mission.launch \
  execute_mission:=true \
  calibration_confirmed:=true \
  enable_servo_drop:=true \
  show_window:=true
```

所有节点就绪后，终端会显示：

```text
Type 'FLY' to authorize ARM/TAKEOFF:
```

最后一次确认飞机和场地安全，然后输入：

```text
FLY
```

只有输入完全正确的 `FLY`，飞机才会解锁和起飞。

## 六、MAVROS 已经启动时

如果其他终端已经运行 MAVROS，使用下面的命令，避免重复启动：

```bash
roslaunch cuadc_vision autonomous_two_drop_mission.launch \
  start_mavros:=false \
  execute_mission:=true \
  calibration_confirmed:=true \
  enable_servo_drop:=true \
  show_window:=true
```

## 七、相机和检测节点也已经启动时

如果 MAVROS、`camera_node.py` 和 `detector_node.py` 都已经运行：

```bash
roslaunch cuadc_vision autonomous_two_drop_mission.launch \
  start_mavros:=false \
  start_vision:=false \
  execute_mission:=true \
  calibration_confirmed:=true \
  enable_servo_drop:=true
```

启动前可以检查关键话题：

```bash
rostopic list | grep -E "mavros/state|local_position/pose|vision/yolo/detections|camera_info"
```

## 八、如何查看任务状态

另开一个已经加载 ROS 环境的终端：

```bash
rostopic echo /autonomous_drop/status
```

查看飞机与目标之间的三维距离：

```bash
rostopic echo /autonomous_drop/target_distance_m
```

查看目标相对飞机的 X/Y/Z 偏移：

```bash
rostopic echo /autonomous_drop/target_offset_xyz
```

常见状态含义：

| 状态 | 含义 |
|---|---|
| `TAKEOFF` | 正在起飞到 2.5 m |
| `SEARCH_FORWARD` | 正在向前搜索目标 |
| `AIM_DESCEND` | 正在下降并修正 X/Y |
| `ALIGN_HOLD` | 已进入误差范围，正在进行 2 秒计时 |
| `DROP` | 正在执行 A 或 B 投放 |
| `SHIFT_FOR_B` | A 完成，正在向前移动 0.08 m |
| `CLIMB_RTL` | 正在升高到 3 m |
| `RTL` | 正在请求自动返航 |
| `PILOT_OVERRIDE` | 遥控器已经接管 |

## 九、紧急停止和人工接管

飞行中出现异常时：

1. 立即用遥控器切换出 `GUIDED`，例如切换到已经验证过的 `LOITER`、`POSHOLD` 或其他人工控制模式。
2. 脚本检测到模式变化后，会停止发送位置目标，不会强行切回 `GUIDED`。
3. 终端按 `Ctrl+C` 可以停止 ROS 脚本，但飞行中应优先使用遥控器接管。

不要把关闭终端当作唯一的紧急处理方式。

## 十、常见问题

### 1. ROS 找不到 launch 文件

```bash
cd ~/catkin_ws
catkin_make --pkg cuadc_vision
source /opt/ros/noetic/setup.bash
source devel/setup.bash
rospack find cuadc_vision
```

### 2. 飞控没有连接

检查：

```bash
rostopic echo /mavros/state
```

如果没有数据，确认飞控 USB、串口和 MAVROS。

### 3. 相机或检测没有数据

检查：

```bash
rostopic hz /vision/color/image_raw
rostopic echo /vision/yolo/detections
```

### 4. 脚本提示 `calibration_confirmed=false`

这是安全保护。必须确认坐标方向、相机高度、搜索距离和舵机参数后，才能使用：

```text
calibration_confirmed:=true
```

### 5. 舵机对应错误

当前只是暂定：

```text
A = CH5
B = CH6
```

确认实际接线后，修改：

```text
autonomous_drop_mission/config/mission.yaml
```

中的：

```yaml
servo_a_channel: 5
servo_b_channel: 6
servo_open_pwm: 1500
servo_close_pwm: 1000
```
