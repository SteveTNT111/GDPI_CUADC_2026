# CUADC 自主瞄准投放改造提示词

请在现有 ROS Noetic `cuadc_control` 半自动投放测试链路上，实现可控、可恢复的
自主桶目标瞄准与投放状态机。不得因为一次视觉丢帧、短时检测失败或深度无效就
直接结束任务；目标丢失应转换为悬停、升高重捕获和局部搜索流程。飞控连接、解锁、
GUIDED 模式、本地位姿或安全边界失效时，仍必须停止任务并优先交还 LOITER。

实现要求：

1. 保留人工半自动模式，并增加显式 `autonomous_mode`。自主模式的启动命令即视为任务
   授权：检查 armed、LOITER 和本地位姿后自动进入 GUIDED；尚未看到目标时从当前位置
   升高搜索，目标连续稳定后才允许水平接近。半自动模式仍需 Enter，且等待期间目标
   失效时应回到等待状态，而不是退出 launch。
2. 使用低空稳定优先的分层瞄准：默认 2.0m 粗对准、1.7m 精对准、1.5m 最终
   投放。只有水平误差、高度误差和机体速度稳定满足当前阶段阈值后才允许下降，
   目标丢失时禁止继续下降；重捕获默认不高于 2.0m。
   控制高度与 D435 画面中心深度必须明确区分；在没有现场最佳深度标定值前，
   不得把相机中心深度直接当作 MAVROS 本地 Z 使用。
3. 对桶的绝对 NED 位置进行低通滤波和跳变拒绝。桶是静止目标，控制器应跟踪稳定的
   世界坐标，而不是逐帧追逐原始深度噪声。
4. 视觉目标丢失后执行分级恢复：立即保持当前位置；超过短时窗口后继续悬停；随后升到
   重捕获高度，以最后目标位置为中心执行逐渐扩大的局部方形搜索。重新检测到目标后恢复
   粗对准；只有超过可配置搜索超时才结束任务。
5. 明确区分消息停更、未检测到桶、视觉/深度无效、A/B NED 无效和目标坐标跳变，并在
   日志中输出恢复状态。
6. setpoint 应持续高频发布，保持当前航向，限制单次水平和垂直位置指令距离，避免原始
   目标跳变形成激烈动作。
   `/mavros/state` 短暂漏心跳时应冻结阶段、保持当前位置并给予有限恢复宽限；只有
   收到新鲜非 GUIDED 状态或超过宽限时间才结束任务。
7. 最终投放除 X/Y 和高度误差外，还必须检查飞机水平、垂直速度；所有条件默认连续稳定
   `1.0s` 后立即触发舵机。
8. A/B 流程应保留；自主模式启用 B 时，A 完成后保持 GUIDED 并继续 B，全部完成后再切回
   LOITER。提供可选的“B 必须选择不同目标”距离黑名单。
9. `ground_test=true` 继续保持原有拆桨手持测试行为，不切 GUIDED、不发布飞行 setpoint。
10. 更新 launch 参数和 `ctrl_README.md`，说明状态机、默认参数、启动方式、丢失恢复行为、
    已知边界和本次修改记录；完成 Python 语法检查与 ROS 包构建验证。

不在本次改造中重写 YOLO、相机驱动或舵机底层节点。若现有消息无法区分“看见桶但深度
无效”和“完全未检测”，先在控制侧给出准确日志与保守恢复行为，并把像素射线—地面求交、
多目标轨迹 ID 和舵机执行确认列为后续接口升级项。
