##此文件夹储存了2025年备赛的全部代码，特点是使用了ROS1和Ubuntu20.04，是2026年备赛的坚实基础##


命名格式以codex为标准（x为数字，表示阅读顺序）
后期代码过于冗余，存在大量无用代码，建议在code1的基础上，写自己的代码。

code1在apm固件环境下实现自动飞矩形。1.0这一系列都是在此基础上，验证其他东西的。

code1.1的目的是验证eigen自动坐标系转换,这一部分重点看go_to_pub函数中对朝向的限制（第70行）和坐标变换部分（249行），坐标变换的目的是保证每次走点都是按起飞时飞机朝向的角度，走预期的点。如果不变换，走点就是按东北天走的。

code1.2加上了yaml获取参数，相当于每次运行代码会在外面的文件获取走点坐标以及其他参数。这样方便调试，以及改变策略。如果直接把参数写在代码里，每次改完都要重新编译，很麻烦。yaml文件放在config文件夹里。

code1.3将除起飞以外的时间判断改为距离判断走点切换（第312行）。原来用的状态切换都是靠计时的。但是实际飞行过程中，很难把控飞机多长时间会到达预期位置，因为我们比赛场地很大。采用期望位置和当前位置的差距作为切换条件，很好。但是也需要我们合理设定差值的误差范围，同时也必须引入超时强制切换，这个我们后面再谈。

code2 主要是用来测试舵机控制的。具体的PWM需要实际测试。取决于舵机摇臂安装角。

code3 在apm固件环境下实现视觉自动投靶。实现了视觉的首次引进。这里重点看视觉发布的信息，是如何订阅并应用到代码的。重点信息为引导向量，即相机中心指向目标圆中心的一段水平向量。这里需要关注视觉那边的坐标系和你这边的xy是否一致，需要两人沟通。

code4 全流程代码一代，这里假设投放点已知。这份代码很重要，需要仔细品味。前面验证的一些东西开始走向函数化。这里的GO_TO_TAR_1，PUT_AIM_1，PUT_1，三状态分工的模式，一直延续到了最终版。TAR_1的坐标为前面粗略确定的桶的坐标。AIM为精准瞄准。后面主要围绕着TAR_1的精度和AIM的精度进行改良代码。

code4.1，同样是很重要的代码，这里桶的位置未知。因此新建状态TAR_FIND ,FIND 的过程粗略确定三个不同直径的桶的位置。

code4.2,主要修改同样是TAR_FIND ，增加了已知桶数量不定的情况下，决定投桶顺序的逻辑。很关键。

code4.2.1,对AIM状态进行改良，增加TAR位置看不到圆的情况处理。这一份代码的思路是把上一次识别到圆的位置保存下来，若当前位置没有识别到圆，则使用上一次识别到圆的位置。想法很美好，但是实际效果一般，会反复横跳，我认为可能是我们调的不够好。后面又采用了新的看不见圆处理方案。

code4.2.2，写了一份在投靶区不找桶直接投放，进侦察区。是因为发现实在是投不进去（后面换机载电脑才可以），因此考虑多得时间分。后来意识到只需要修改强制切换状态的判断时间即可，此版也就没啥用。

code5，北部赛区使用代码。相比之前有所修改，增加了降落引导。北部赛区调机现场，基本确定了投不进去的事实，放弃了投靶。这份代码也很重要，虽然放弃了投靶，但是一些思考还是很有必要的。首先是意识到了为啥一直投不进去的原因，图像存在延迟，因此确定的桶的位置始终不太对。打算采用限制飞行速度来消除延迟的影响，效果不佳；对于AIM阶段看不见圆的情况做了全面的分析，新增better_pos,始终记录引导向量最小的位置，如果看不见圆或者圆的引导向量模长越来越大，就回到better_pos,但是这个效果也不佳，私以为是很好的思路，建议继续研究。

code6,单独验证投放瞄准部分。区赛后发愤图强，开始专攻TAR_FIND，GO_TO_TAR，PUT_AIM三个重要状态。发现飞机两点距离较大时，行动过程飞机姿态变化太大，且速度较大，影响目标定位。因此决定多设立几个点，来限制速度和姿态（手动微分这一块），确实投进了，当然也是刘博老师帮助调参的结果。其实有更好的方法实现这一目的，直接在gotopub函数中进行限制，但是笔者愚钝。

code7,最终全流程版本1。瞄准阶段加入低通滤波（对传感器信息的预处理），这里尝试通过飞机当前的姿态角，来计算桶的位置，消除姿态偏差的影响（因为姿态偏移，摄像头不朝正下方了，计算出的桶的位置并非实际的位置），效果不佳。对于AIM看不见桶的情况，采用升高来扩大视野的方式。

code8，对 经纬度发点方式的验证。发现飞机坐标变换后的坐标出现偏差（原因是飞机在地面记录的方向角和起飞后不一致，起飞后会干扰罗盘）。因此采用测出场地实际经纬度，来确定位置，这样就跳过了罗盘的信息，只用到了RTK的信息。

code9，最终全流程版本2(经纬度绝对位置版)。对经纬度进行了进一步的完善，在远距离发点使用经纬度，防止偏移。瞄准阶段仍采用原发点方式，保证飞机方向不变，确保视觉融合不出问题。
