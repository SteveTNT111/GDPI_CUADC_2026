## 一、第一次：让 GitHub 变成中文 + 克隆仓库

### 1.1 浏览器装 Tampermonkey

Edge 或 Chrome 打开 https://www.tampermonkey.net/ → 点"下载" → 安装扩展。
在edge 浏览器扩展直接搜篡改猴

### 1.2 装两个翻译脚本（两个都要装）

- https://greasyfork.org/zh-CN/scripts/435208-github-%E4%B8%AD%E6%96%87%E5%8C%96%E6%8F%92%E4%BB%B6
- https://greasyfork.org/zh-CN/scripts/407485-github-internationalization

每个链接打开 → 点"安装此脚本"。装完后 GitHub 全中文。

### 1.3 注册 GitHub 账号

打开 https://github.com → 右上角"注册" → 填邮箱、密码、用户名 → 验证邮箱。

**把用户名发给伍尚京。** 他会把你加进仓库。

### 1.4 VS Code 克隆仓库

```
Ctrl+Shift+P → 输入 git clone → 回车
→ 粘贴 https://github.com/SteveTNT111/GDPI_CUADC_2026.git
→ 选保存位置（放 D 盘根目录）
→ 克隆完成后右下角点"打开"
```

以后每次打开 VS Code，直接 `文件 → 打开文件夹 → 选这个文件夹`。

```

```

```

```

## 四、文件命名

| 类型 | 格式 | 示例 |
|------|------|------|
| Python 脚本 | `功能描述.py` | `camera_node.py` |
| Launch 文件 | `功能.launch` | `run_main.launch` |
| 说明文档 | `代码文件名.md` | `servo_test.md` |
| 配置文件 | `模块.yaml` | `params.yaml` |
| 测试记录 | `日期_内容.md` | `2026-07-02_后山试飞.md` |

**禁止**：`最终版.py`、`新版本.py`、`改.py`、`副本.py`

## 五、交付标准

每个代码文件必须配一个**同名 .md 说明文件**，至少写清楚：

- 这个脚本干什么
- 依赖什么（pip install xxx）
- 怎么跑（命令行）

### 正确示例

```
代码/陈智勇/
├── yolo_detect.py      # YOLO 检测脚本
└── yolo_detect.md      # 说明：干什么、依赖、怎么跑
```

### 不要这样

```
代码/陈智勇/
├── test.py
├── test2.py
├── 最终版.py
└── （没有说明文件）
```

## 八、常见问题

**Q：我点错了，加了别人的文件怎么办？**
在"暂存的更改"里找到那个文件 → 点 `−` 号取消暂存。

**Q：推送失败说"不是最新的"？**
有人比你早推送了。先 `Ctrl+Shift+P → git pull`，再重新推送。

**Q：仓库被我搞乱了怎么办？**
不要慌，不要乱点。在群里喊伍尚京。

## 九、禁止做的事

- ❌ `git add .` / VS Code 里点"全部暂存"
- ❌ 碰 `cuadc_src/` 里面的文件
- ❌ 把代码打成 zip 发到群里
- ❌ `git push -f`（强制推送）
- ❌ 新建文件夹做版本管理（`v1/`、`最终版/`）—— Git 就是干这个的

## 十、为什么定这些规则

不是因为"规范很重要"这种空话。

- 视觉组之前交了 306 个 .py 文件，没人知道哪个是主节点
- 控制组写了舵机脚本，视觉组不知道在哪、怎么用
- 交付靠压缩包，版本靠文件夹名
- 有人回家之后代码在电脑上，别人没法接手

**规范不是为了好看，是为了你不在的时候别人能接着干。**
