## 1. GPS 协议与刷新率

**先分清两件事**：「用什么协议」和「刷新率多少」是独立的——协议决定"怎么发"，刷新率决定"发多快"。

### 协议：UBX vs NMEA
- **NMEA**：ASCII 文本协议（`$GPGGA,...`），人可读但啰嗦、占带宽。
- **UBX**：u-blox 的**二进制**私有协议，紧凑、信息全（原始伪距、精度因子等），飞控（ArduPilot/Betaflight）会自动优先用它。穿越机 GPS 基本都走 UBX。
- NMEA 也能跑 10Hz，只是二进制 UBX 在同波特率下更容易撑高刷新率。

### 雷迅(CUAV) 工业级 GPS 刷新率

| 模块 | u-blox 芯片 | 导航刷新率上限 |
| --- | --- | --- |
| NEO-3 系列（含 NEO-3X） | M9N（单频） | 最高 25Hz（常用 5–10Hz） |
| C-RTK 9P / 9Ps | ZED-F9P（双频 L1/L2 RTK，厘米级） | 最高 20Hz（全星座满 RTK 常配 5–8Hz） |

> "9P/9Ps"的名字来自 **F9P** 芯片。落地前建议对一下雷迅官方 datasheet 的 update rate 一栏确认默认值。

### 关键点：飞控不会跑到芯片上限
- ArduPilot `GPS_RATE_MS` 默认 **200ms = 5Hz**，可改 100ms=10Hz，再高意义不大。
- 原因：**EKF 是"低频 GPS 修正 + 高频 IMU 递推"**。位置/速度这种慢漂移量靠 GPS 5–10Hz 修正就够，快速动态全靠上千 Hz 的 IMU 撑。**GPS 刷新率从来不是瓶颈。**

## 2. IMU 刷新率与如何调整

IMU（陀螺+加速度计）才是"快"的那一层。**调法在 BF 和 AP 里完全不同，别照搬。**

### 穿越机（Betaflight/INAV）——就是你说的"陀螺 4k、8k"
- 在 Configurator 里直接设 **Gyro update frequency** + **PID loop frequency**，如 `8k/8k`、`4k/4k`、`2k/2k`。
- "8kHz" = 陀螺每秒被读 8000 次 + PID 环每秒算 8000 次，越高延迟越低、手感越跟手。
- **受陀螺芯片上限约束**：
  - MPU6000 / ICM-20602：陀螺 8kHz
  - ICM-42688-P：可达 32kHz
  - BMI270：约 3.2kHz（所以很多 BMI270 板子只能 3.2k/3.2k）
- 加速度计一般更慢（常见 1kHz）：姿态控制主要靠陀螺，加速度计管长期姿态基准。

### 自主载机（ArduPilot）——没有"设 8kHz PID"这个概念
- **主控制环** `SCHED_LOOP_RATE`：Copter 默认 **400Hz**（可 800）。这才是 PID 环频率，不是几千 Hz。
- **IMU 快速采样** `INS_FAST_SAMPLE`（位掩码）：让选定 IMU 以 8kHz（或芯片上限）采样。**目的不是提高 PID 频率**，而是：
  - **谐波陷波滤波（Harmonic Notch）**：抗电机振动，需要高频原始陀螺数据；
  - **抗混叠**：高频采样再降采样，减少振动混叠进控制环。
- **滤波截止**：`INS_GYRO_FILTER`（默认 ~20Hz 低通）、`INS_ACCEL_FILTER`。
- 所以 AP 里调的是"IMU 采 8kHz + 控制环 400Hz + 陷波滤波"，而非 BF 的"8kHz PID 环"。

## 3. BF 与 AP PX4 的控制环区别

**一句话**：BF 是"单环速率控制器"（为手动飞行、极低延迟）；AP/PX4 是"多环级联控制器"（位置→速度→姿态→速率，为自主飞行）。

### Betaflight（穿越机 / Acro）：单环
```
摇杆(期望角速率) ──▶ [速率 PID] ──▶ 电机混控 ──▶ 电调
                        ▲
                   陀螺(实测角速率)
```
- 核心只有**一个环：速率环（Rate PID）**，PID 作用在**角速率**（deg/s）误差上。
- Angle/Horizon 模式只是外面加一层"角度误差→P→期望角速率"，本质仍是速率环。
- **没有位置/速度/高度控制器**（GPS Rescue 是外挂补丁）。位置由飞手的眼和手来闭环。

### ArduPilot / PX4（自主载机）：四级级联
```
目标位置 ─[位置P]─▶ 目标速度 ─[速度PID]─▶ 目标倾角/加速度 ─[姿态P]─▶ 目标角速率 ─[速率PID]─▶ 电机混控
   ▲慢环~50-100Hz       ▲                    ▲中环                    ▲快环 400Hz~1kHz
  估计位置            估计速度               估计姿态                  陀螺
        └───────── 全部状态来自 EKF 融合(IMU+GPS+气压+磁罗盘+光流…) ─────────┘
```
- **四层级联**：一层输出是下一层目标。最内的速率环和 BF 的速率环干同一件事，只是外面套了三层自主控制。
- 需要完整 **EKF2/EKF3** 状态估计，控制器作用在"估计出来的状态"上。

### 对比表

| 维度 | Betaflight | ArduPilot / PX4 |
| --- | --- | --- |
| 控制结构 | 单环（速率）+ 可选角度外环 | 位置→速度→姿态→速率 四级级联 |
| 设计目标 | 手动 Acro，极低延迟、跟手 | 自主飞行、定点、航点、offboard |
| PID 作用对象 | 主要是角速率 | 每级都有：位置P、速度PID、姿态P、速率PID |
| 状态估计 | 轻量互补滤波（只为姿态/OSD），无位置估计 | 完整 EKF，估计位置/速度/姿态 |
| 速率环频率 | 4k–8kHz（=整个主环） | AP 400Hz（可 800）/ PX4 ~1kHz |
| 为什么这个频率 | 小、轻、高带宽，压榨延迟求手感 | 大轴响应慢，400Hz 足够；算力花在估计与滤波 |
| 位置来源 | 飞手眼睛（GPS Rescue 除外） | GPS/RTK/SLAM/光流 → EKF |
| 抗振滤波 | 动态陷波 + RPM 滤波（DShot 回传转速） | 谐波陷波（需 INS_FAST_SAMPLE 高频采 IMU） |
| 失效保护/自主 | 极简 | 完整（geofence、failsafe、模式逻辑） |

### AP 与 PX4 之间
同一家族（级联 + EKF），差异在实现：
- **PX4**：模块化（`mc_rate_control`/`mc_att_control`/`mc_pos_control`），四元数姿态控制器，速率环 ~1kHz。
- **ArduPilot**：`AC_AttitudeControl` + `AC_PosControl`，输入整形（加速度/角速率限幅），速率环跑主环 400Hz。
- 你发 setpoint 时两者在 MAVROS 层被统一，感知不到这层差别。

## 4. 落到我们的机队

- **CUADC/电赛载机**（Pixhawk/匿名跑 AP 或 PX4）：你发**位置/速度 setpoint**，喂进的是**最外面的位置/速度环**，底下三层飞控自己闭——这就是"只发坐标飞机就会自己飞过去"的原因。
- **穿越机**（BF）：没有位置环，飞手拨杆直接进速率环，纯手动。不能"发个坐标让它自己去"，除非上 GPS Rescue 这类简化功能。
- **为什么载机不追求 8kHz**：AP 400Hz 是级联里最内一环，大轴够用；算力优先给 EKF 估计与陷波滤波（IMU 仍 8kHz 采样喂滤波）。
- **为什么 GPS 只要 5–10Hz**：EKF 靠 IMU 补高频，GPS 只做慢漂移修正。

> 一句话：**BF 把闭环交给飞手，只做最内速率环、拉满频率求手感；AP/PX4 把闭环交给自己，用 EKF + 四级级联实现自主，速率环够用即可。**
