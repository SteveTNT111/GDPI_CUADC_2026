# 系统程序架构

> 从相机拍到舵机投放，整个链路的节点划分、话题定义、状态机设计。
>
> 关联：[[00_备赛计划]]、[[01_队伍分工]]

---

## 整体架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                       状态机 (State Machine)                      │
│  巡航 → 检测到桶 → 瞄准 → 投放 → 侦察 → 返航                     │
│  根据预设程序 + 桶状态 + 检测置信度 切换                          │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 状态切换指令
┌────────┐   ┌──────────┐   ┌──────────┐   ┌──────────────┐
│ 相机    │   │ 视觉检测  │   │ 融合解算  │   │ 底层控制     │
│ D435i  │→ │ YOLO/CV  │→ │ 坐标变换  │→ │ PID + MAVROS │→ 飞控
│ 节点    │   │ + 6D位姿  │   │ + 误差计算 │   │ setpoint     │
└────────┘   └──────────┘   └──────────┘   └──────────────┘
                              ↑
                        ┌──────────┐
                        │ RTK/GPS  │
                        │ 机身位姿  │
                        │ (/mavros) │
                        └──────────┘
```

## 节点划分（共 5 个 ROS 节点）

### 1. 相机节点 `camera_node.py`

**负责**：驱动 D435i，发布彩色图和深度图

| 发布话题 | 类型 | 说明 |
|---------|------|------|
| `/camera/color/image_raw` | `sensor_msgs/Image` | 彩色图 |
| `/camera/depth/image_raw` | `sensor_msgs/Image` | 深度图 |
| `/camera/depth/points` | `sensor_msgs/PointCloud2` | 点云（可选） |

**实现**：`pyrealsense2` → ROS 桥接

---

### 2. 视觉检测节点 `vision_node.py`

**负责**：检测目标（圆筒/替代标识物），输出相对相机的位置和置信度

**方法**（双保险）：
- **主方案**：传统 CV 黄色圆检测（陈智勇已部署上机 ✅）
- **备方案**：YOLOv8 检测（赖斯远负责，等圆筒到了训）
- **上位姿估计**：如果 (x,y,z) 精度不够，上 PnP 解 6D 位姿

| 订阅话题 | 说明 |
|---------|------|
| `/camera/color/image_raw` | 彩色图输入 |
| `/camera/depth/image_raw` | 深度图输入 |

| 发布话题 | 类型 | 说明 |
|---------|------|------|
| `/vision/target_pose` | `geometry_msgs/PoseStamped` | 目标相对相机的位置+朝向 |
| `/vision/target_confidence` | `std_msgs/Float32` | 检测置信度（0~1） |

**关键接口**（潘科宇 + 控制组对接）：
```python
# vision_node 发布的 topic 结构
pose = PoseStamped()
pose.header.frame_id = "camera_link"
pose.pose.position.x = 0.5    # 目标在相机前方 0.5m
pose.pose.position.y = -0.2   # 目标偏左 0.2m
pose.pose.position.z = 0.0
# orientation 可选，用于 6D 位姿
```

---

### 3. 融合解算节点 `fusion_node.py`

**这是郑老师说最难的环节**——把视觉结果 + 飞机自身位姿 + RTK 坐标融合成控制误差信号。

**负责**：
1. 订阅 `/vision/target_pose` → 拿到"目标相对相机"
2. 订阅 `/mavros/local_position/pose` → 拿到"飞机在哪儿"
3. 订阅 `/mavros/global_position/global` → 拿到 RTK 经纬度
4. 坐标变换：相机系 → 机体系 → 地面系（或 ENU 系）
5. 算出"目标相对飞机的水平误差 (dx, dy) + 高度误差 (dz)"

| 订阅话题 | 说明 |
|---------|------|
| `/vision/target_pose` | 视觉检测结果 |
| `/mavros/local_position/pose` | 飞机机身位姿 |
| `/mavros/global_position/global` | RTK 经纬度 |

| 发布话题 | 类型 | 说明 |
|---------|------|------|
| `/control/target_error` | `geometry_msgs/Vector3` | (误差_x, 误差_y, 误差_z) |

**坐标系说明**：
```
camera_link → base_link (tf 树，相机装在飞机上的外参)
base_link → map (由 MAVROS 给，local_position)
    或者
base_link → global (由 MAVROS 给，RTK)

最终输出：目标在 map 系下的 (x, y, z) 或 (dx, dy, dz) 误差
```

### 4. 底层控制节点 `control_node.py`

**负责**：接收误差信号 → PID 解算 → 通过 MAVROS 发 setpoint 给飞控

| 订阅话题 | 说明 |
|---------|------|
| `/control/target_error` | 融合解算后的误差信号 |

| 发布话题 | 说明 |
|---------|------|
| `/mavros/setpoint_position/local` | 位置 setpoint（往哪飞） |
| `/mavros/setpoint_velocity/cmd_vel_unstamped` | 速度 setpoint（怎么飞过去） |

| 服务调用 | 说明 |
|---------|------|
| `/mavros/cmd/arming` | 解锁/上锁 |
| `/mavros/set_mode` | 切模式（GUIDED / AUTO / RTL） |

#### Setpoint 是什么？

**setpoint 就是"你让飞机飞到哪/怎么飞"的指令。**

| Setpoint 类型 | 发什么 | 适合场景 |
|-------------|--------|---------|
| **位置 setpoint** | `PoseStamped(x, y, z)` — "飞到 (5, 2, 10) 这个点" | 航点巡航、飞到桶上方 |
| **速度 setpoint** | `TwistStamped(vx, vy, vz)` — "以 2m/s 向东飞" | 瞄准微调、悬停时纠偏 |
| **混合用** | 先位置飞到附近，再速度微调对准 | **推荐的瞄准策略** |

**AP Guided 模式下**：
- 位置 setpoint 发一次就行，飞机自己飞过去（比 PX4 Offboard 省心）
- 速度 setpoint 需要持续发
- 没有 PX4 的 500ms watchdog 限制

```
瞄准策略：
1. 发位置 setpoint → 飞到桶正上方（高度保留 5-8m 安全余量）
2. 切换速度 setpoint → 缓慢下降 + 水平对准
3. 误差 < 阈值 → 发投放信号
```

**PID 控制框架**（后续细化参数）：

```python
class PIDController:
    def __init__(self, kp, ki, kd):
        self.kp = kp
        self.ki = ki
        self.kd = kd
        self.integral = 0
        self.prev_error = 0

    def compute(self, error, dt):
        self.integral += error * dt
        derivative = (error - self.prev_error) / dt
        self.prev_error = error
        return self.kp * error + self.ki * self.integral + self.kd * derivative
```

### 5. 状态机节点 `state_machine_node.py`

**负责**：整个任务的顶层调度，决定飞机当前处于什么模式

**状态定义**：

```
          ┌───────────┐
          │  TAKEOFF  │  ← 起飞
          └─────┬─────┘
                ↓
          ┌───────────┐
          │   CRUISE  │  ← 按预设航点巡航，视觉持续检测
          └─────┬─────┘
                ↓  (置信度 > 阈值 && 稳定跟踪)
          ┌───────────┐
          │  APPROACH │  ← 飞向目标，逐步降低高度
          └─────┬─────┘
                ↓  (误差 < 对准阈值)
          ┌───────────┐
          │   AIMING  │  ← 精确对准，速度微调
          └─────┬─────┘
                ↓  (对准完成)
          ┌───────────┐
          │  DROP_BOMB│  ← 投放第一个桶
          └─────┬─────┘
                ↓  (投放完成)
          ┌───────────┐
          │  AIMING2  │  ← 对准第二个桶（如果还有）
          └─────┬─────┘
                ↓
          ┌───────────┐
          │ DROP_BOMB2│
          └─────┬─────┘
                ↓  (两个桶都投完)
          ┌───────────┐
          │   RECON   │  ← 自动进入侦察航线，快速完成
          └─────┬─────┘
                ↓  (侦察完成)
          ┌───────────┐
          │    RTL    │  ← 返航降落
          └───────────┘
```

**状态转换触发条件**：

| 转换 | 触发条件 |
|------|---------|
| CRUISE → APPROACH | YOLO/CV 置信度 > 阈值（如 0.7）且持续 N 帧 |
| APPROACH → AIMING | 水平误差 < 1m，高度 < 5m |
| AIMING → DROP_BOMB | 水平误差 < 0.3m，高度 < 3m，稳定 > 2s |
| DROP_BOMB → AIMING2 | 第一个桶已释放，且还有第二个桶 |
| DROP_BOMB → RECON | 所有桶已释放 |
| RECON → RTL | 侦察航点全部完成 |

## 话题总览

| 话题 | 方向 | 类型 | 频率 |
|------|------|------|------|
| `/camera/color/image_raw` | camera → vision | `sensor_msgs/Image` | 30Hz |
| `/camera/depth/image_raw` | camera → vision | `sensor_msgs/Image` | 30Hz |
| `/vision/target_pose` | vision → fusion | `geometry_msgs/PoseStamped` | 10-30Hz |
| `/vision/target_confidence` | vision → state_machine | `std_msgs/Float32` | 10-30Hz |
| `/control/target_error` | fusion → control | `geometry_msgs/Vector3` | 10-30Hz |
| `/mavros/local_position/pose` | MAVROS → fusion | `geometry_msgs/PoseStamped` | 20Hz |
| `/mavros/global_position/global` | MAVROS → fusion | `sensor_msgs/NavSatFix` | 5Hz |
| `/mavros/state` | MAVROS → control | `mavros_msgs/State` | 2Hz |
| `/mavros/setpoint_position/local` | control → MAVROS | `geometry_msgs/PoseStamped` | 10-20Hz |
| `/mavros/setpoint_velocity/cmd_vel_unstamped` | control → MAVROS | `geometry_msgs/TwistStamped` | 10-20Hz |
| `/state_machine/current_state` | state_machine → 所有 | `std_msgs/String` | 状态变更时 |

## 部署方案

**三台 NUC 分工**（基于实际 WiFi 状况动态调整）：

```
NUC 黑色（控制组）← WiFi 有问题，先拉网线或换机子
  ├─ fusion_node.py
  └─ control_node.py

NUC 白色 #1（视觉组）
  ├─ camera_node.py    ← D435i 物理连接在这台
  └─ vision_node.py

NUC 白色 #2（仿真组 / 伍尚京）
  ├─ state_machine_node.py
  └─ 仿真环境 (AP SITL)
```
