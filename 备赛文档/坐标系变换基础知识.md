## 1. 总览 一条链贯穿全流程

视觉识别到的桶，要一路变换成飞控能执行的坐标，数据流是这样的：

```
像素(u,v)
  │ 反投影(+深度+内参)
相机光学系 RDF (x右 y下 z前)
  │ 静态 TF 外参(相机怎么装)
ROS 机体系 base_link FLU (x前 y左 z上)
  │ 姿态四元数旋转 + 加飞机位置
ROS 世界系 ENU (x东 y北 z上)
  │ ←── mavros 自动双向翻译 ──→
飞控机体系 FRD (x前 y右 z下) / 飞控世界系 NED (x北 y东 z下)
  │ 借 EKF 原点经纬高
WGS84 / GEO (纬度 经度 高度)
  │ 横墨卡托投影
UTM (Easting Northing 米) → MGRS (字符串)
```

**记忆钩子**：ROS 侧（相机除外）都是"左/上/东"这套右手系（**FLU / ENU**）；飞控侧都是"右/下"这套（**FRD / NED**）；相机单独是 **RDF**；GEO=WGS84；UTM/MGRS 是地图层，不进实时控制回路。**mavros 负责把 ROS 侧(ENU/FLU) ↔ 飞控侧(NED/FRD) 来回翻译。**

## 2. 逐个坐标系详解

### 2.1 像素坐标系（2D 图像）
- **轴**：`u`=图像列、`v`=图像行；原点在图像**左上角**。
- **增大方向**：u 往**右**增大、v 往**下**增大。只有二维，无深度。
- **中心**：主点 (cx, cy) 来自相机内参。
- **→ 相机光学系**：需要深度 Z 与内参(fx,fy,cx,cy)，针孔反投影：
  `X=(u−cx)·Z/fx，Y=(v−cy)·Z/fy，Z=深度`（即 detector_node 的 project_pixel）。

### 2.2 相机光学系 RDF
- **轴**：X=右、Y=下、Z=前（光轴射出、看出去的方向）；原点在光心。
- **增大方向**：X 往**右**、Y 往**下**、Z 往**前/远**（越远深度越大）。
- **别名**：RDF（Right-Down-Forward）。ROS 里叫 `*_optical_frame`（如 `d435i_color_optical_frame`）。
- **→ base_link**：靠**静态 TF 外参**（相机安装位置+朝向）。我们的下视安装（相机在重心正下方、镜头朝下、顶部朝机体前方）对应：
  平移 `(0, 0, −h)`（h=相机到 base_link 垂直距离），旋转四元数 `(x,y,z,w)=(0.7071, −0.7071, 0, 0)`。
  ```
  rosrun tf2_ros static_transform_publisher  0 0 -0.08  0.7071 -0.7071 0 0  base_link  d435i_color_optical_frame
  ```
  （若 realsense 驱动已发布相机内部 TF，则把此变换发到相机根 frame `d435i_link`，避免光学系有两个父）。

### 2.3 ROS 机体系 base_link FLU
- **轴**：X=前、Y=左、Z=上；原点≈质心/飞控。
- **增大方向**：X 往**前**、Y 往**左**、Z 往**上**。
- **别名**：FLU（Forward-Left-Up），ROS REP-103 机体约定，mavros/TF 用它。
- ⚠️ **这是做相机→机体变换时真正进入的机体系，不是飞控的 FRD！**
- **→ ENU**：用飞机当前姿态四元数（`/mavros/local_position/pose` 的 orientation，即 base_link 在 ENU 中的姿态）把机体向量**旋到 ENU**，再加飞机的 ENU 位置。

### 2.4 ROS 世界系 ENU
- **轴**：X=东、Y=北、Z=上；原点=EKF 原点/上电起飞点。
- **增大方向**：X 往**东**、Y 往**北**、Z 往**上**。
- **谁在用**：`/mavros/local_position/pose` 读的、`/mavros/setpoint_position/local` 发的都是它，单位米。
- **→ 飞控 NED**：**mavros 自动转**，`N=ENU.y, E=ENU.x, D=−ENU.z`（XY 对调、Z 取反）；机体侧同步 FLU↔FRD。

### 2.5 飞控机体系 FRD
- **轴**：X=前、Y=右、Z=下；原点=质心。
- **增大方向**：X 往**前**、Y 往**右**、Z 往**下**。
- **谁在用**：AP/PX4 固件内部、MAVLink 机体系消息。
- **与 base_link(FLU) 关系**：X 相同，**Y、Z 取反**（左↔右、上↔下）。

### 2.6 飞控世界系 NED
- **轴**：X=北、Y=东、Z=下；原点=EKF 原点。
- **增大方向**：X 往**北**、Y 往**东**、Z 往**下**（越低值越大）。
- **谁在用**：**AP 和 PX4 内部都用 NED**、`SET_POSITION_TARGET_LOCAL_NED` 线上。
- **→ WGS84**：借 EKF 原点的经纬高，把"北/东 米"换成经纬度差：`north≈Δlat·111320m`，`east≈Δlon·111320·cos(lat)`，`down=−Δalt`。

### 2.7 WGS84 与 GEO
- **轴**：纬度 lat、经度 lon、高度 alt；是**椭球面角度坐标**，非平面直角。
- **增大方向**：lat 往**北**、lon 往**东**、alt 往**上**增大。
- **谁在用**：RTK/GPS 输出、`SET_POSITION_TARGET_GLOBAL_INT`（lat/lon×1e7）、`MAV_FRAME_GLOBAL`。
- **说明**：AP/MAVLink 里的 **GEO（geographic/global）就是 WGS84 经纬高**——同一基准、同一套角度坐标。地面站地图上显示的经纬度即它。

### 2.8 UTM
- **是什么**：把地球按经度切成 60 个 6° 带（zone），带内投影成**平面直角**，坐标是**米**：`(Easting 东距, Northing 北距) + 带号 + 半球`。
- **增大方向**：Easting 往**东**增大（含 500000m 假东距，恒为正）、Northing 往**北**增大。
- **谁在用**：大范围**米制平面**计算与地图显示。**mavros `global_position/local` 内部就是先把经纬度投影成 UTM 再算本地 ENU/map。**
- **来自**：WGS84 经横轴墨卡托投影（按经度定 zone）。

### 2.9 MGRS
- **是什么**：在 UTM 之上编码成字母数字串，如 `50TMK1234567890` = 带号 + 100km 方格字母 + 格内东距/北距。
- **精度**：数字位数决定，10 位=1m。
- **谁在用**：人读/军用/制图；部分地面站（Mission Planner）可显示。
- **来自**：由 UTM 编码（UTM 又来自 WGS84）。

## 3. 总表速查

| 坐标系 | 维度 | 轴（物理 / 增大方向） | 谁用 | 变到下一个 |
| --- | --- | --- | --- | --- |
| 像素 | 2D | u→右，v→下 | 图像/YOLO | +深度&内参反投影 |
| 相机光学系 RDF | 3D | X右 Y下 Z前 | 视觉/PnP/深度 | 静态TF外参→base_link |
| ROS机体 base_link FLU | 3D | X前 Y左 Z上 | mavros/TF/代码 | 姿态四元数旋转→ENU |
| ROS世界 ENU | 3D | X东 Y北 Z上 | local_position / setpoint | mavros自动→NED（XY换、Z反） |
| 飞控机体 FRD | 3D | X前 Y右 Z下 | AP/PX4内部、MAVLink | 与FLU差：Y、Z取反 |
| 飞控世界 NED | 3D | X北 Y东 Z下 | AP/PX4内部、LOCAL_NED | 借EKF原点→WGS84 |
| WGS84 / GEO | 全球 | lat北 lon东 alt上 | RTK/GPS、GLOBAL_INT | 横墨卡托投影→UTM |
| UTM | 投影平面 | Easting东 Northing北（米） | 地图/mavros内部 | 编码→MGRS |
| MGRS | 字符串 | 东距/北距数字→东/北增 | 制图/GCS | — |

## 4. 常见误区纠正

1. ❌「AP 用 ENU、PX4 用 NED」→ ✅ **AP 和 PX4 内部世界系都是 NED**。ENU 只是 **mavros 在 ROS 侧转出来的呈现**。MAVLink 线上也是 NED。
2. ❌「相机外参配到飞控 FRD 机体系」→ ✅ 相机→机体的 TF 进入的是 **ROS 的 base_link，是 FLU（前左上）**，不是飞控 FRD。配外参、看 body_x/y/z 符号都按 **FLU**。
3. ❌「NED 是机体坐标系」→ ✅ **NED 是世界系**；机体系是 FRD。两者都是飞控侧。
4. ❌「相机是前右下」→ ✅ 相机光学系是 **RDF（右下前）**：X右、Y下、Z前。
5. 右手系：ENU / NED / FRD / FLU **全是右手系**；左手系只在游戏引擎/部分可视化里出现。

## 6. 室内定位与 EKF 对齐

- 室内无 GPS 时，需要给飞控喂一个位置源，位置/速度 setpoint 才能用；否则飞控没有水平位置估计，Loiter/GUIDED 会被拒绝。
- 常见室内位置源：光流+测距、VIO、SLAM（FAST-LIO2，输出 ENU，原点=雷达初始化位置）、AprilTag→`vision_pose`。
- 喂法：`/mavros/vision_pose/pose` → MAVLink `VISION_POSITION_ESTIMATE` → EKF3 融合（参数 `EK3_SRC1_POSXY=6` ExternalNav；PX4 对应 `EKF2_AID_MASK`）。
- **EKF 自动对齐**：飞控收到外部位姿后，会把 local 原点/朝向对齐到该位姿（首帧 reset + yaw_offset 估计），磁罗盘方向定义被覆盖，室内磁罗盘参与度≈0。**你的上层代码不需要手动处理对齐**。
- 各定位源给飞控的格式：GPS=NMEA、测距=`DISTANCE_SENSOR`、视觉/SLAM=`VISION_POSITION_ESTIMATE`。
- 竞赛策略（哈工大 code9）：**远距离发 WGS84（抗罗盘漂移）、近距离发 ENU（保机头稳、利于视觉）**，两种混用。
