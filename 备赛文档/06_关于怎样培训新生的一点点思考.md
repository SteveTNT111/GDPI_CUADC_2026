# 关于怎样培训新生的一点点思考

> 去年（2025.09）培训大一新生的时候，我只会告诉他们去浏览器打开腾讯元宝网页版——因为当时腾讯元宝提供的免费 DeepSeek 推理比官网还好一点。现在回头看，那个建议虽然在当时有效，但它只回答了"用哪个工具"，没有回答"怎么用好 AI"。

---

## 〇、实验室分组优先级与定位（先定架构，再谈培训）

在讨论"怎么培训"之前，必须先把"培训什么人、往哪个方向"想清楚。以下是我们实验室的实际情况和定位：

### 核心原则：嵌入式 > 机械

我们**不是 RoboMaster 战队**，不需要设计复杂的机器人结构。画一个简单的四轴机架就够了。

| 方向          | 为什么是这个优先级                                                               |
| ----------- | ----------------------------------------------------------------------- |
| **嵌入式（电控）** | 让一个会嵌入式的人去学简单 CAD 建模——没难度。让一个只会建模的人去学嵌入式/控制/视觉——几乎不可能。**嵌入式是底座，机械是上层。** |
| 控制 + 视觉     | 两个方向都基于 Python + ROS，底层相通，可以交叉学习                                        |
| 机械          | 够用就行，Fusion 360 画个机架、出个 3D 打印件                                          |

### 分组定义（不是"电控"，是"嵌入式"）

如果只是打电赛，控制和嵌入式是分开的。但我们实验室的实际方向划分应该是：

| 分组 | 学习路径 | 适合 |
|------|---------|------|
| **嵌入式组** | ESP32 → STM32 → 外设驱动 → 通信协议 → 飞控底层 | **零基础新生**，专心学好嵌入式，不要跳来跳去 |
| **控制组** | Python → ROS → MAVROS → 路径规划 → 飞控二次开发 | 有一定编程基础，可以快速跳过嵌入式阶段 |
| **视觉组** | Python → OpenCV → YOLOv5/v8 → 模型部署 → 推理实战 | 有编程基础，跟控制组底层相通 |
| **机械组** | Fusion 360 → 3D 打印 → 简单机架设计 | 对结构感兴趣，能配合嵌入式组装飞机 |

> [!tip] 视觉和控制为什么底层相通
>
> 两个方向都依赖 Python + ROS。视觉组的模型推理结果（目标检测框、位姿估计）最终要发给控制组做决策。如果一个人同时懂视觉和控制，价值远大于两个分离的岗位。

### 培训分层

```
零基础新生          → 嵌入式组（专心打基础，不去碰视觉/ROS）
有编程经验          → 可跳过嵌入式，直接进控制组或视觉组
嵌入式基础 + 想进阶 → 横向学视觉或控制（Python 基础已有）
```

### 目标

招到下一届队员之后，**在我毕业之前培养出 10 个有参赛水平的核心组成员**，能够独立备战明年的 CUADC 和电子设计竞赛。

## 一、去年的教训

### 做了什么

- 让新生用腾讯元宝网页版 → 免费 DeepSeek，比官网快
- 本质上：**把 AI 当搜索引擎用**——遇到不懂的，丢进去问

### 问题在哪

1. **免费模型效果差**——不管包装得多好，底层是阉割过的，回答深度不够
2. **只教了工具，没教方法**——新生学会了"在元宝里搜问题"，但不会写 prompt、不会追问、不会判断 AI 说的是对是错
3. **工具一年就过时**——腾讯元宝的最佳窗口期只有几个月，现在 DeepSeek 官网更强了，元宝的优势没了

> [!warning] 核心教训
>
> **不要教产品名，要教方法论。** 产品名会变成去年的腾讯元宝，但"怎么让 AI 给你好答案"是永久的技能。

## 二、今年的核心思路

### 分层培养，不搞一刀切

大一新生的水平差距极大——有人连终端是什么都不知道，有人已经自己搭过 Docker。一刀切的方案要么让新手掉队，要么让进阶者无聊。

```
第一层：零门槛入门（第1-2周）
    ↓  会用 AI 聊天、提问
第二层：学会"调教"AI（第3-4周）
    ↓  理解 prompt、上下文、角色设定
第三层：把 AI 嵌入工作流（持续）
    ↓  代码补全、笔记+AI、本地模型
大二+：Agent、CLI 工具、本地部署
```

### 教方法论，不教产品名

按优先级排序，新生最需要学会的：

| 优先级 | 能力              | 说明                                                                       |
| :-: | --------------- | ------------------------------------------------------------------------ |
|  1  | **Prompt 三段式**  | 角色 + 任务 + 约束条件。例："你是一个嵌入式 C 语言助教，帮我解释这段 ESP32 的 GPIO 配置代码，用大一新生能听懂的话"    |
|  2  | **追问与纠偏**       | AI 第一次回答不满意，追问"能更简单一点吗"、"能举个例子吗"、"你确定这个寄存器地址是对的？查一下数据手册"                 |
|  3  | **多模型对比**       | 同一道题分别问 DeepSeek、通义千问、豆包，对比回答——培养批判性思维                                   |
|  4  | **判断 AI 是否在编造** | 嵌入式领域 AI 幻觉特别多——寄存器地址、引脚编号、外设配置经常凭空捏造。核心原则：**AI 给的代码和数据，必须对照数据手册或实际跑一遍** |

## 三、2026 年推荐的新生 AI 工具矩阵

### 零门槛（浏览器/App）

| 工具 | 推荐理由 | 免费 |
|------|---------|:--:|
| **DeepSeek 官网** | 2026 年的 V3/R1 比 2025 年强太多，直接官网用 | ✅ |
| **通义千问 App** | 阿里的，和嵌入式生态（乐鑫/ESP32）资料涵盖更全，支持语音对话 | ✅ |
| **豆包** | 字节系，手机端体验好，碎片时间随时问 | ✅ |

> 不用再绕道腾讯元宝了——2026 年各家官网模型已经足够好。

### API 方案（第三周要用）

训练到第三周配 VS Code + Continue 时，需要 API Key。**不要用 OpenAI 官方 API**——`api.openai.com` 在国内被墙，ChatGPT Plus 会员也不等于有 API 额度（两套独立计费系统）。直接用国产免费 API：

| API | 免费额度 | 国内直连 | 注册地址 |
|-----|---------|:--:|------|
| **DeepSeek API** | 注册送 500 万 tokens | ✅ | platform.deepseek.com |
| **Kimi API** | 新用户送免费额度 | ✅ | platform.moonshot.cn |
| **通义千问 API** | 有免费额度 | ✅ | dashscope.aliyun.com |
| **豆包 API** | 有免费额度 | ✅ | volcengine.com |

> [!warning] 为什么不让老师开 OpenAI Key
> 
> (1) 需要挂梯子、(2) 账单全算老师身上、(3) Key 泄露被盗刷、(4) 多人共用触发风控封号。国产 API 零成本零代理，够新生用到能自己赚钱买更好的。

### 进阶（配进工作流）

| 工具 | 场景 | 门槛 |
|------|------|:--:|
| **Codex** | 终端 AI 助手——直接说话就能让 AI 读代码、写代码、调试、读文档。**今年的新生神器。** 2025 年 9 月它还不存在，今年已经是标配 | 低 |
| **Obsidian + AI 插件** | 笔记整理、知识库检索 | 中 |
| **Ollama + Open WebUI** | 理解"本地推理"和"云端 API"的区别；可离线使用 | 中 |
| **Claude / ChatGPT 桌面版** | 长文档分析（Datasheet、手册）、复杂推理 | 低 |

> [!tip] 为什么 Codex 替代了 Continue
> 
> 2025 年 9 月培训新生时，Codex 还没做出来，当时的最佳方案是 VS Code + Continue + API Key。现在 Codex 已经成熟——新生不需要懂 API Key、不需要装插件、不需要理解 Agent 概念。打开终端，说话，AI 就会读你的代码、查你的文档、帮你写程序、帮你调试。
> 
> 一个从没碰过开发板的高中毕业生 → 在 Codex 的辅助下 → 几周内变成能读 Datasheet、能写嵌入式代码、能排查硬件问题的学生科研助理。
> 
> **这还只是过了一年。**

### 暂不推荐给新生的（留到大二）

| 工具 | 为什么现在不推 |
|------|---------------|
| VS Code + Continue | Codex 出来之后，对新生来说 Continue 的意义已经不大。Codex 覆盖了 Continue 的所有场景，而且不需要配 API Key |
| OpenClaw | 开发者 Agent 编排框架，需要理解 session、worktree、多 Agent 协作 |
| Hermes Agent | 需要"养成"1-3 个月才有价值，新生自己的学习习惯都还没定型 |

## 四、嵌入式方向的特殊提醒

做 CUADC / 无人机 / 嵌入式开发的新生，有一些 AI 特有的坑和甜点：

| ✅ AI 擅长的（多问） | ❌ AI 容易翻车的（要验证） |
|---------------------|--------------------------|
| 解释已有代码逻辑 | 生成底层寄存器操作代码（地址经常编造） |
| 调试日志分析 | 电路/硬件设计建议（AI 不懂模拟电路） |
| C 语言语法/编译报错排查 | 实时性/时序相关的问题（AI 没有硬件直觉） |
| ROS 节点模板、launch 文件格式 | MAVLink 消息的具体字段值（版本差异大） |
| 从 Datasheet 提取摘要 | 不看 Datasheet 直接让 AI 生成驱动代码 |
| 学习路径规划、概念对比 | 竞赛策略/具体得分点判断（AI 没比过赛） |

> [!tip] 培训时可以做的事
>
> 拿一段 ESP32 的 GPIO 初始化代码，故意让 AI 生成一个有寄存器地址错误的版本，新生自己跑一遍 → 硬件不工作 → 对照数据手册排查 → 发现 AI 的错。
>
> **一次实战比讲十遍"AI 有时会编造"更有效。**

## 五、培训大纲建议

### 第一周：认识 AI 助手（1 小时）

- [ ] 什么是大语言模型（10 分钟，用白话讲，不讲 Transformer）
- [ ] 现场演示：同一个问题，不同问法，不同答案
- [ ] 每人注册一个 DeepSeek 账号
- [ ] 作业：用 AI 解释一段最基础的 C 代码（Hello World 级别的），明天回来汇报 AI 说得对不对

### 第二周：Prompt 即编程（1 小时）

- [ ] 三段式 prompt 教学（角色 + 任务 + 约束）
- [ ] 实战：让 AI 当"C 语言助教"，解释 `static`、`volatile`、`const` 的区别
- [ ] 多模型对比练习：DeepSeek vs 通义千问，谁解释得更清楚？
- [ ] 作业：用 AI 写出 ESP32 点灯代码，拿去跑通，记录过程中 AI 犯了哪些错

### 第三周：AI + 开发工具链（1 小时）

- [ ] **Codex 安装与入门**（替代去年的 VS Code + Continue 方案）
- [ ] 现场演示：在终端里让 Codex 读一段 ESP32 代码、解释逻辑、修改功能
- [ ] 演示 Codex 读 Datasheet：把 PDF 丢给它 → 提取关键信息 → 生成驱动代码
- [ ] 作业：用 Codex 辅助完成一个小功能，对比有/无 Codex 的效率差异

> 2025 年 9 月这一周的内容是"装 Continue + 配 API Key"。2026 年的新生不需要懂任何这些——Codex 已经帮他们跨过了所有技术门槛。

### 第四周：AI 辅助项目学习（1 小时）

- [ ] 怎么用 AI 读文档：Datasheet 丢给 AI 做摘要
- [ ] 怎么用 AI 学新概念：SCL/SDA → I2C 总线 → 整个外设体系，一层层追问
- [ ] "什么时候不该用 AI"——AI 帮你理解概念，但替你思考 = 你什么都没学到
- [ ] 讨论：这四周里，AI 帮到你了什么？坑到你什么？

## 六、新生入门硬件最低成本方案

> 有些新生刚刚上大学才买到第一台轻薄本，有些家里连买电脑的钱都紧张。这套方案的目标：**用最少的钱，接入世界上最先进的模型推理 + 入门无人机/嵌入式/机械。**

### 6.1 手机（AI 终端）

- **二手 Redmi / 小米 5G 手机**，预算 500-800 元，能装 Obsidian + Syncthing-Fork
- 手机不需要大存储——多出来的文件用 Syncthing 同步到电脑上
- 手机上跑 Obsidian 写笔记 → Syncthing 同步到电脑 → 电脑上 AI 分析
- 也就是你现在用的这套方案，新生可以直接复制

> 手机 + Syncthing + Obsidian = 最便宜的 AI 移动终端。起步成本：一部能用的二手手机。

### 6.2 电脑

- **二手办公主机 / 旧笔记本**，预算 600-700 元
- 最低要求：能装 Win10 或 Win11、能跑浏览器、能装 VS Code
- 如果动手能力强，捡垃圾拼一台台式机更划算
- 电脑上配好 VS Code + Continue + 国产免费 API → 代码补全和 AI 问答全搞定

### 6.3 无人机入门

两条路，看动手能力和预算：

| 路线 | 起步成本 | 适合 |
|------|---------|------|
| **自制路线** | 开源电路板 + 零件自组 | 动手能力强，想深入理解原理 |
| **航模入门** | 100 元以内遥控器 + 有刷小穿越机 / 纸飞机航模 | 零基础，先飞起来培养兴趣 |

### 6.4 嵌入式入门

- 从简单的嵌入式项目开始：ESP32 点灯 → GPIO → I2C → 传感器
- 开发板几十块钱，传感器几块钱一个
- 配合 AI（DeepSeek/通义千问）写代码、排查问题

### 6.5 机械入门

- 软件：**Fusion 360**（对学生免费）
- 从简单建模开始：画一个零件 → 3D 打印出来 → 组装
- 3D 打印机：实验室共享一台，耗材一卷几十块

### 6.6 关于个人装备

- 开发板、烙铁、万用表等基础工具——**自己掏钱买**
- 实验室提供的是：机架、飞控、NUC、深度相机等贵重公共设备
- 不要指望薅实验室的现成零件，自己买才知道珍惜

## 七、实验室成员分层与筛选

### 7.1 三层成员体系

```
         核心组（科创突击队/人工智能与自主系统小组）
         ├── 能独立参赛、能技术传承、能做技术积累
         ├── 嵌入式/控制/视觉至少通一个方向
         └── 这是你重点培养的对象，目标是毕业前出 10 人
                    │
         应用组（梯队队员）
         ├── 对无人机有兴趣但技术敏感度不够
         ├── 航测应用小组、飞行表演小组、植保小组、设备管理小组
         ├── 承担下山科普、免费航测、领导参观表演等任务
         └── 可旁听培训，有兴趣可以升核心组
                    │
         混分组
         ├── 对技术没兴趣，只为混比赛加分
         └── 安排他们负责：电池充电、管钱管文档、保养设备、打扫卫生
```

### 7.2 考核标准（不能比去年更松）

去年的考核只需要：
- EDA 打样一张自己画的 PCB
- SolidWorks 画一个圆柱体
- 组装一架无人机（不需要飞起来）

→ 通过之后成了"正式成员"就没事干了。**这不够。**

> [!note] 新的考核底线
>
> 作为一个无人机应用专业的学生、实验室的科研助理，**学会无人机组装调试应该是最基本的**——不是需要学长手把手教的"技能"，而是你自己应该主动掌握的底线。指望加入"组装调试小组"让学长一步步教——这种心态本身就说明不适合核心组。

### 7.3 关于"组装调试小组"

以前单独开一个组装调试组，结果大部分人都往这挤——因为电控看起来太难了。但：

> [!quote]
>
> 让一个会嵌入式的人去学简单 CAD 建模——没难度。让一个只会建模的人去学嵌入式/控制/视觉——几乎不可能。

所以**取消独立的"组装调试组"**。组装调试是每个人的基本要求，不是某个组的专属。所有的培训路径都以嵌入式/控制/视觉为起点，机械建模是附加技能。

### 7.4 关于会费（无人机协会并入后）

如果无人机协会并入实验室，新成员每人交 **50 元会费**。累计起来够买一大批开发板和耗材来培训他们。

### 7.5 什么样的人不要

| 类型 | 表现 | 处理 |
|------|------|------|
| 为混奖而来 | 对技术无兴趣、无追求 | 安排杂活，不投入培训资源 |
| 等喂型 | 等着学长手把手教，不主动学 | 给一次警告，不改就劝退 |
| 通过考核就躺平 | 去年那种——过了入门考核就啥也不干 | 考核只是开始，不是终点 |

## 八、写在最后

> 去年我告诉新生"去浏览器打开这个网址"，今年我想告诉他们"学会让 AI 成为你的第二大脑"。
>
> 这之间的差距不是我变聪明了，是我在 2025 年 5 月到 2026 年 7 月这一年多里，从"跟风本地部署 DeepSeek 跑在 CMD 里不知道能干嘛"，到"配 Docker、搭 Open WebUI、写 Skill、理解 Agent 架构"——走完了自己的认知跃迁。
>
> 新生不需要一开始就走到我现在的水平，但他们需要一个清晰的路标：**你现在在第几层、下一步是什么、哪些坑不要踩。**
>
> 这个文档就是那个路标的第一版。
