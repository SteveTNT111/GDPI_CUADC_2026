🎯 **竞赛策略梳理**（7/7，基于哈工大代码分析和讨论）：

**全程 GUIDED 模式方案：**

不用地面站 KMZ 航线，不用 AUTO 模式来回切换。全程由小电脑在 GUIDED 模式下自主发送航点：

```
起飞 → 向前飞 30m → 投放区（投 2 个桶）
  → 桶数量=0 → 向前飞 20m → 侦察区（升高 + 识别 3 个标志物）
  → 返航
```

每个阶段都是：`发 ENU setpoint → 等到达 → 发下一个 setpoint`

**时间约束分析：**
- 比赛总时长 5 分钟（300 秒）
- 按 AI 估算，完成任务后必须在 **75 秒内**返航降落，否则飞行时长分会倒扣降落分
- 如果返航太慢 → 主动放弃降落分，先停飞行时长分

**坐标系基础知识**（今天做了大量梳理）：

| 坐标系 | 原点 | 轴 | 用途 |
|--------|------|-----|------|
| **ENU = local** | 起飞点 | E=东, N=北, U=上 | setpoint 发这个！单位米 |
| **机体 FRD** | 飞机质心 | F=前, R=右, D=下 | 相机→机体的变换 |
| **WGS84** | 地球椭球 | 纬度/经度/海拔 | RTK GPS 输出 |

- 室内 SLAM 无人机也是 ENU 坐标系（原点=上电位置，磁罗盘给方向，SLAM给位置）
- `/mavros/setpoint_position/local` 发的就是 ENU，不是机体坐标
- GUIDED 模式下发一次 setpoint 就够了，飞控自己规划飞过去（AP 没有 watchdog，PX4 有 500ms）
- "密室模式"= Mission 模式（AUTO航点），我们不用这个，全程 GUIDED

🛡️ **安全验证脚本 `safe_flight_test.py` 已就绪**（7/7）：

放在 `cuadc_src/scripts/safe_flight_test.py`，设计原则：**绝对安全**

```bash
# 用法：手动起飞后，切 GUIDED，代码发 setpoint 往前飞 2 米验证
rosrun cuadc_vision safe_flight_test.py _test_distance:=2.0 _test_direction:=north
```

**安全机制：**
- ❌ 不自锁、不自起飞（遥控器手动操作）
- ✅ 检测到 GUIDED 模式后才开始发 setpoint
- ✅ 到达目标后自动切 LOITER 交还控制权
- ⚠️ **紧急切断：遥控器 CH5 切 LOITER → 飞机立刻不理电脑指令，原地悬停**

**⚠️ 绝对不要切 ALTHOLD 或 STABILIZE — 不保定位置，风吹就漂！**

**飞行验证步骤：**
1. 遥控器 LOITER 模式手动起飞到 5-10m
2. 终端启动 `safe_flight_test.py`
3. 遥控器切 GUIDED → 观察飞机是否往北飞 2 米
4. 到达后自动切 LOITER → 手动降落
5. 任何时候不对劲 → 立刻切 LOITER

🔧 **ZD550 机架 STEP 模型导入——踩了一个三小时的大坑**

在闲鱼上花两块钱买了 ZD550 的 STP 三维模型（俄罗斯模友用 KOMPAS-3D 导出的 STEP AP242），用来在装配体里量孔位、设计定制转接件。

结果被 SOLIDWORKS 坑了整整三个小时——来回重复导入 → 另存为 → 关掉重开 → 零件全部丢失 → 再导入 → 再来一次。

**根因**：STEP 导入后零件是"虚拟零件"——住在装配体文件里，没有独立的 `.sldprt` 物理文件。点"另存为 + 包括所有参考零部件"时，SOLIDWORKS 去找这些零件的文件 → 不存在 → 全部报"未找到"。关了再打开，装配体全是空的，画的图纸全没。

**最终解决**：**文件 → Pack and Go**，勾选"包括工程图"和"包括模拟结果"（虽然不知道这个机架有什么工程图和模拟结果，但勾上没坏处），全选保存到新文件夹，虚拟零件全部硬拆成独立 `.sldprt`。关掉重开 → 零件全在 ✅。

**踩坑教训**：

> 不管是程序还是工程上，遇到那种重复出现的问题，不应该尝试去逃避它，或者重复操作指望它能自己消失。应该**尽早定位问题**，把它解决掉。

这适用于一切——SolidWorks 的导入机制、代码编译报错、飞控参数不生效——但凡发现自己在做同一件事第三次第四次，就该停下来问：我是不是在撞墙？墙是什么？怎么绕？

今晚花了三小时验证了一个道理：**重复操作不会让 bug 消失。停下来，查原因，十分钟能省三个小时。**

🔧 **补充：STP 底板草图复制到自制零件**

从 STP 导入的底板孔位要复用到自己画的转接板/挂架上，完整流程：

1. 在目标零件里新建一个基准面（和底板平行）
2. 在 FeatureManager（设计树）里同时按住 **Ctrl**，选中源草图和这个基准面
3. **插入 → 派生草图（Derived Sketch）**——整张草图包括实线、构造线、辅助线、尺寸全部拷过来
4. 如需独立修改：右键派生草图 → "解除派生"（Underive）

> 之前只知道转换实体引用，但它只拷实线，构造轴和辅助线带不过来。派生草图才是搬整张图的正确操作。搞了半天才弄明白 Feature Manager 就是设计树。

**GitHub 代替 Obsidian Sync**：多端协作之后发现，GitHub 本身就是最好的笔记同步工具。手机端装一个能执行 git 命令的工具（代替 VS Code 的 Git 生态位），Obsidian 读取本地仓库，pull/push 全走 Git。大文件和归档只留电脑本地，需要同步的 md 笔记放在独立的子仓库里推送。

**队员要不要强制用 Obsidian**：目前他们没有用，只用 VS Code 加乱七八糟的 Markdown 预览插件——我以前也是这么过来的。武老师给我们弄了五六个独立屏幕，完全可以 Obsidian 在左边看笔记、VS Code 在中间写代码、Claude Code 提要求——双屏协同。但我还没有决定要不要强制推下去。

**开发模式变了**：以前是台式机上写代码推给 NUC。现在是直接让 NUC 上的 Claude Code 改工作空间里的代码，改完 NUC 推回 GitHub，台式机只用来读笔记和看代码。台式机的"单机强大"不再是必需品。

**NAS 与算力去中心化**：一直在追求电脑单机的强大——硬盘接满、强劲 CPU/GPU、24h 不关机、Docker 跑 immich/n8n。现在意识到：只要路由器网速够，低功耗 NAS 才是正确答案。主机只需要 1-2TB 的 NVMe——够装工作软件和下几个游戏就行。多出来的大容量存储全扔 NAS 里，主机体积反而可以做小。硬盘集中到 NAS 做存储池 + RAID 备份，Docker 容器（immich 相册、Jellyfin 动漫、n8n 自动化）全跑在 NAS 上。台式机只接必要硬盘，手机 128G 就够——所有照片和文件实时备份到 NAS。电脑坏了硬盘坏了都不怕数据丢。

**好处延伸到家人**：我妈的照片也能放上去，不用插移动硬盘翻来翻去。有些回忆不方便随时读取，但如果能在手机和电脑上随时访问本地 NAS 的云备份，那就完全是另一回事了。

### 队长笔记：关于 AI、专科生和"科技平权"

> 以下不是进度，是一个人在深夜的思考。记录下来是因为这些东西决定了队伍怎么走。

说实话，如果高考成绩好一点，进了更好的队伍，或者进了 RM 队伍，我应该会有队长或者老队员来带我。但在这里，没有比我更厉害的，没有人可以带我。

我不否认——如果是在别的队伍里，我可能只是一个梯队成员。但在这里，全部东西都要靠我自己想，全部的推进、全部的发现都要由我来把控。

我没有把这些经历拿来炫耀，或者拿来贬低别人。我一直在想：如果让我重新来一次，但只花 500 块钱、300 块钱，甚至 100 块钱——通过买一些小工具、零件、开发板，或者是充高效的 token 来辅助学习，而不是靠在 B 站看劣质视频、看空谈广告来试错受挫——我能不能凭借自己踩过的坑，让别人少走点弯路？

因为我感觉 AI 现在越来越恐怖了。他们认为学了东西结合 AI 就能搞定一切，但实际上 AI 让很多人越来越难找到工作。问题是——有没有什么办法，能让 AI 对他们进行科技平权？

我不否认有些人本来就很有钱，有没有 AI 对他们的生活影响都不大。但说实话，现在这个时期，AI 真的太恐怖了。我们正在经历一个颠覆的时代——谁能越熟练地掌控 AI、使用 AI 来完成原本需要花很多时间的那种费时费力的工作，谁就能拉低专科生与研究生之间的差距。

**所以我给队伍定的方向是**：不追求从零造轮子。用 AI 扫清英语障碍（GitHub 翻译脚本）、用 AI 加速代码开发（Claude Code）、用 AI 降低学习门槛——把省下来的时间，用在只有人才能做的事情上：判断方向、做决策、飞飞机、打比赛。

今晚发现了哈工大和哈工大的 CUADC 开源仓库。看了一遍——我们一个广东山里没人听过的大专，靠自己看视频、自己琢磨出来的状态机思路和视觉反推流程，和国防七子的方向走的是同一条路。

现在觉得，培养组员掌握某一项特定工程技能，不如培养他们**终身学习的能力**，以及对无人机的兴趣。过去一些很好的职业，几年内可能就被淘汰了。医生也许还能维持得久一点——我总不能指望给自己看病的时候医生问 ChatGPT "这什么病"——但理工科不一样。

以后我招梯队成员，要有两个硬指标：
1. **对无人机的兴趣 + 对技术的追求**
2. **工程素养 + 提示词能力**——这是基本功，没有这两样，光有兴趣也只是外行人
